How it works

  • Option 1 — IAM Access Keys: Create an IAM user with programmatic access, paste the keys above. Persistent, no expiration until rotated.
  • Option 2 — Temporary Credentials: From SSO or assume-role, paste access key + secret + session token. These expire and need to be refreshed.
  • Fallback: If no credentials are saved here, jobs fall back to your local AWS config (~/.aws/config, SSO cache, environment variables).
